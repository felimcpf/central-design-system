# Design System: Figma to Code — Design Spec

**Date:** 2026-04-09
**Goal:** Enable a non-technical user to build a full React design system from Figma and use it to generate themed product pages — without writing code.

---

## Overview

A shared React component library is extracted from Figma using the Figma MCP integration in Claude Code. The library supports 4 product-specific color themes (mapped from Figma variable modes). Storybook documents all components with a live theme switcher. A sync script keeps the codebase in sync with Figma going forward. Product pages are generated by describing them in plain English to Claude Code.

---

## Architecture

Three phases:

### Phase 1 — One-time Setup (with Claude Code)
1. User provides Figma file URL and API token.
2. Claude Code reads the Figma file via Figma MCP.
3. Claude extracts:
   - All design tokens: 4 color variable modes (one per product), plus shared base tokens (spacing, typography, border radius).
   - All published components from the main Figma file (the user specifies which Figma file page contains the component library).
4. Claude generates the full codebase (see File Structure below).
5. User runs `npm run storybook` to preview the result.

### Phase 2 — Ongoing Sync (one command)
- When Figma changes, user runs `npm run sync-figma`.
- The script calls the Figma REST API, re-fetches variables and component data, regenerates the 4 theme token files and CSS variable files.
- No Claude involvement needed for routine syncs.

### Phase 3 — Building Product Pages (plain English)
- User tells Claude: "Build a [page type] for [Product N] using the design system."
- Claude assembles the page from existing components, wrapped in the correct ThemeProvider.
- User previews in browser, gives feedback, Claude adjusts.

```
│   ├── product-1.json         # Color scale for Product 1
│   ├── product-2.json         # Color scale for Product 2
│   ├── product-3.json         # Color scale for Product 3
│   └── product-4.json         # Color scale for Product 4
├── src/
│   ├── themes/
│   │   ├── product-1.css      # CSS variables generated from token JSON
│   │   ├── product-2.css
│   │   ├── product-3.css
│   │   ├── product-4.css
│   │   └── ThemeProvider.tsx  # Wraps app; accepts product name prop
│   └── components/
│       └── [ComponentName]/
│           ├── [ComponentName].tsx         # React component using CSS vars
│           └── [ComponentName].stories.tsx # Storybook story
├── scripts/
│   ├── sync-figma.js          # Fetches latest tokens/components from Figma
│   └── generate-themes.js     # Transforms token JSON → CSS variable files
├── .storybook/
│   └── preview.tsx            # Theme switcher toolbar (all 4 products)
└── package.json               # npm scripts: storybook, sync-figma, build
```

## Theme System

- All React components reference color values exclusively via CSS custom properties (e.g. `var(--color-primary)`). No hardcoded colors in component code.
- Each `product-N.css` file defines the full set of CSS variables for that product's color scale.
- `ThemeProvider.tsx` accepts a `product` prop (`"product-1"` | `"product-2"` | `"product-3"` | `"product-4"`), loads the corresponding CSS file, and wraps children.
- Switching themes = changing the `product` prop. Zero component code changes.
- Storybook `preview.tsx` adds a toolbar dropdown to toggle themes live for all stories.

---

## Figma MCP Integration

- Tool: Figma MCP plugin configured in Claude Code settings with a Figma personal access token.
- On setup: Claude uses the MCP to read the Figma file, extract variable modes and component definitions.
- On sync: `sync-figma.js` calls the Figma REST Variables API directly (no Claude needed) to pull updated token values and write them to `tokens/product-N.json`.
- Figma variable mode names map directly to product theme names.

## Non-Technical User Workflow

| Task | What the user does |
|------|-------------------|
| Initial setup | Install Claude Code, paste Figma token into settings, provide Figma URL to Claude |
| Generate design system | Tell Claude: "Read my Figma file and generate the design system" |
| Preview components | Run `npm run storybook` in Claude Code terminal |
| Sync after Figma changes | Run `npm run sync-figma` in Claude Code terminal |
| Build a product page | Tell Claude: "Build a homepage for Product 2 with hero, features, CTA" |
| Review / iterate | Preview in browser, describe changes to Claude in plain English |

---

## Storybook Setup

- Every component has a `.stories.tsx` file generated alongside it.
- Storybook is pre-configured with a global theme switcher in the toolbar (all 4 products).
- All variants/states of each component are shown per story.
- Can be deployed (e.g. to Chromatic or Netlify) as a shared reference for designers.

---

## Out of Scope

- Deployment pipeline for product pages (handled separately per product).
- Authentication, routing, or backend logic for product apps.
- Figma component variants beyond what is defined in the Figma file.
- Adding new design tokens not present in Figma (all tokens must originate in Figma).
